    热备机环境自动化检测

    转至元数据结尾
    • Created and last modified by yushaolong on 九月 25, 2017
    转至元数据起始

    问题现状

    由于服务器访问流量短时间内增幅较大，导致目前线上机器无法支撑大流量访问，需要热备机上线，基于现有业务，一台服务器达到热备机的状态并上线，需要如下操作：

    1.从镜像创建新主机 , 授权mysql访问(增、删、改、查)@op 
    2.从master克隆代码@QA 
    3.热备机环境检测,包括nginx，php-fpm，php扩展与线上一致，api可访问 
    4.定时任务 (1).日志切割脚本@肖辉 (2).errorLog报警@QA 
    5.op上线

    目前对于步骤3热备机的环境检测都是人工操作，所以本文编写了shell脚本，来解决步骤3的自动化检测。

    设计方案

    由于最终需要在开发机A上运行shell脚本，将线上机器B与热备机C的环境比对，并检测热备机C的运行情况。所以总体思路是以开发机做中转，大概流程如下：

     

    功能实现

    实现了10项检测，包括nginx，php，项目等，具体见下图。编码细节在此不再赘述，可以参见脚本 hotcheck.sh

     

    具体的编码实现细节在此不再赘述，可以参见脚本 hotcheck.sh 

    使用说明

    使用前请先打开脚本，编辑配置区域，否则会执行失败 
    脚本提供了几个命令：

    usage: 
    1. hotcheck –start(主要命令) 
    (1)用于检测线上与热备机环境, 若不一致,则同步线上环境 到 热备机环境 
    (2)若项目存在，检测项目配置并初始化，然后请求api检测是否正常工作 
    (3)添加例行任务 
    2. hotcheck –program 
    用于项目初始化,在命令1中项目检测失败时,可以重新执行命令1中第(2)项 
    3. hotcheck –curl 
    用于检测热备机项目api接口是否正常工作，数据库是否已授权 
    4. hotcheck –crontab 
    用于添加例行任务 
    5. hotcheck –clear 
    用于清除本地和远端检测中生成的文件碎片 
    6. hotcheck –step 
    查看上线步骤
